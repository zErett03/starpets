# План тестирования доставки через /test-webhook — ПК

Цель: проверить фикс friendship (3 слоя) на десктопе. Прод: `https://starpets-production.up.railway.app`

> ⚠️ Тест тратит реальные деньги (шаг покупки). Баланс токена ~$16.76 — бери дешёвый оффер.
> ⚠️ `/test-webhook` — это **POST** (через адресную строку браузера не отправить). Остальное — GET, открывается в браузере.

---

## Шаг 0. Подготовка (обязательно)

1. **Задеплоить фикс** — он сейчас только в рабочем дереве:
   ```
   git add -A && git commit -m "fix: re-send friendship for dispatched orders (page + monitor)"
   git push origin HEAD:master
   ```
   Дождаться деплоя Railway (~1–2 мин).
2. **Тестовый Roblox-аккаунт** — ник, который НЕ совпадает с аккаунтом продавца на маркете (иначе риск `NO_ACCESS 210`). Например, `zErett03`. Залогинься в Roblox в браузере заранее.
3. **Выбрать оффер в наличии** — открой `/system-status`, возьми `ggsel_offer_id` активного оффера с дешёвым товаром.
4. Открой **Railway-логи** в соседней вкладке — будем смотреть строки friendship.

---

## Шаг 1. Создать заказ (POST)

В терминале:
```
curl -X POST https://starpets-production.up.railway.app/test-webhook ^
  -H "Content-Type: application/json" ^
  -d "{\"ggsel_offer_id\": <OFFER_ID>, \"roblox_username\": \"<НИК>\"}"
```
(в PowerShell перенос — backtick `` ` ``; в cmd — `^` как выше)

Из ответа записать: **`order_id`** и **`ggsel_order_id`**.
Статус будет `pending` — это норма (`/test-webhook` доставку сам не запускает).

---

## Шаг 2. Запустить доставку (GET, браузер)

```
/trigger-deliver?order_id=<ORDER_ID>
```
Это ставит задачу DELIVER → воркер делает buy → create_trade → friendship(T=0) → `dispatched`.

---

## Шаг 3. Проверить, что заказ ушёл в dispatched (GET)

```
/order-info?order_id=<ORDER_ID>
```
Ожидаем: `delivery_status = dispatched`, заполнены `bot_name` и `starpets_custom_id` (= trade_id).

> Статус смотрим здесь, а НЕ через монитор (монитор слеп к трейду из-за лимита 50 — известная проблема).

Если `failed`/`pending` — стоп, смотрим `error_reason` и логи `[Deliver] ...` (особенно `create_trade FAILED`). Это укажет точку отказа.

---

## Шаг 4. Открыть страницу доставки (GET, браузер) — **держать вкладку активной**

```
/delivery?id=<GGSEL_ORDER_ID>
```
⚠️ Параметр `id` принимает **ggsel_order_id**, не внутренний order_id.

Ожидаем: имя бота, таймер. Страница теперь авто-рефрешится каждые 20с — каждый рефреш повторно шлёт friendship (десктоп-слой). Вкладку держим на переднем плане.

---

## Шаг 5. Принять товар в Roblox

1. В Roblox добавь бота (имя со страницы) **в друзья**.
2. Подожди ~1 мин — бот должен **принять заявку** (это и есть проверяемый фикс: friendship пере-шлётся со страницы и/или из монитора).
3. Открой **профиль бота на roblox.com → кнопка Join** (НЕ телепорт из игры — телепорт даёт Error 773).
4. Зайдя в Adopt Me на сервере бота — **прими входящий трейд**.

---

## Шаг 6. Подтвердить доставку (GET)

```
/order-info?order_id=<ORDER_ID>
```
Ожидаем со временем `delivery_status = done`. Проверь инвентарь в Adopt Me.

---

## Что искать в логах Railway (критерий успеха фикса)

| Лог-строка | Значит |
|---|---|
| `[Deliver] create_trade response ... trade_id=... bot_name=...` | трейд создан |
| `[Deliver] friendship sent` | первый (ранний, T=0) вызов |
| `[delivery] friendship re-sent order_id=... trade_id=...` | сработал слой страницы (на каждом заходе/рефреше) |
| `[MonitorDelivery] order_id=... friendship re-sent (periodic) trade_id=...` | **сработал серверный слой** (каждые 30с, первые 10 мин) |

**PASS:** бот принял заявку в друзья в течение ~1–2 мин, трейд завершился, товар в инвентаре, заказ → `done`.
**FAIL:** зафиксировать шаг, вывод `/order-info` и строки `[Deliver]`/friendship из логов.

---

## Примечание про ПК vs телефон

На ПК активны все три слоя (страница + авто-рефреш + монитор). Это валидирует happy-path и сам фикс. Но клиентские слои (рефреш) работают, только пока вкладка активна — **отдельный тест с телефона** нужен, чтобы подтвердить, что серверный слой (монитор) тащит доставку, когда покупатель уходит из браузера в приложение Roblox.
